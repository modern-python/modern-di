# Design: Docs dedupe

## Summary

A dedupe pass over the MkDocs site applying the Diátaxis placement rule from
the 2026-07-06 verbosity benchmark: every concept gets exactly one canonical
home; every other mention becomes a sentence plus a link. Two stub pages
merge into their real homes (with `mkdocs-redirects` preserving URLs), the
two historical migration guides condense to ~300 words each, and the audit's
padding list is fixed. Target: ~30.6k words → ~22k across 39 pages, with no
information deleted — only de-duplicated.

## Motivation

The benchmark found per-page sizes normal (median 515 words ≈ httpx/click)
but concept duplication severe: the scope/captive-dependency rule is
substantively explained in 8+ pages, the 3.0 warns-then-raises story in ~11
prose locations, the `filterwarnings` recipe in 8 files, a ~150-word
"Framework Context Objects" block copy-pasted across 4 integration pages,
and the two from-X guides share ~500 verbatim words. Duplicates drift
independently (the stale-reference rot already seen in code applies to prose
equally).

## Design

**Canonical homes** (everything else links):

| Concept | Home |
|---|---|
| Scope hierarchy + captive-dependency rule | `providers/scopes.md` |
| Context patterns (framework context objects, non-propagation ❌/✅) | `providers/context.md` |
| Override mechanics | `recipes/testing-overrides.md` |
| 3.0 warns-then-raises + `filterwarnings` recipe | `migration/to-3.x.md` |
| Singleton → `Factory(cache=True)` mapping | `providers/factories.md` |
| Sync-only resolution / async finalizers | `introduction/design-decisions.md` |
| Runtime circular-dependency guard | `troubleshooting/circular-dependency.md` |

**Page merges** (URLs preserved via `mkdocs-redirects`, added to
`docs/requirements.txt` + `mkdocs.yml`): `testing/fixtures.md` →
`integrations/pytest.md` (drops the one-page Testing nav section);
`introduction/that-depends-or-modern-di.md` → a section of
`introduction/comparison.md`.

**Troubleshooting pages all stay** — they serve error-message-driven search.
They keep the error text, immediate fix, and a link to the canonical
explanation; their re-explanations go.

**Historical guides condensed hard**: `migration/to-1.x.md` and `to-2.x.md`
each become ~300 words — the breaking-change list stated once (currently
three times per page), minimal before/after snippets, no marketing prose.

**from-X guides** (`from-that-depends.md`, `from-dependency-injector.md`)
keep their conversion value (API mapping tables, install tabs) but their
framework-integration and pytest sections trim to links into
`integrations/`, and their scope explanations to links into
`providers/scopes.md`; the ~500 verbatim shared words shrink accordingly.

**Padding fixes** from the audit: `factories.md` bullet/section repeat,
`context.md` triple identical sentence, `duplicate-type-error.md`
self-restatement, `about-di.md` trimmed of the feature pitch duplicating
`index.md`/`comparison.md`.

**mkdocs.yml** updates in the same pass: nav (Testing section removed,
merged pages gone), `llmstxt` sections map (entries for removed pages
deleted, descriptions of reshaped pages adjusted), `redirects` plugin map.

## Non-goals

- No new content, no restructuring beyond the two merges, no tone rewrite.
- Integration pages stay standalone in their how-to steps; only the
  copy-pasted concept blocks become links.
- `architecture/` untouched (its halving is the remaining benchmark item).
- Code snippets that readers copy-paste stay complete even where similar
  across pages — dedupe targets prose explanations, not runnable examples.

## Testing

`just docs-build` (mkdocs `--strict`: broken links/anchors/nav fail) green;
redirect map covers both merged URLs; `just lint-ci` green; word-count
before/after reported per page in the PR body; reviewer spot-checks that
each canonical home actually contains what the links promise.

## Risk

- Over-trimming a page that a reader lands on cold (integration pages from
  search) — mitigated by the standalone-how-to non-goal and per-page review.
- Link rot inward: future pages re-inlining concepts — mitigated by the
  canonical-homes table above living in this spec, referenced from the PR.
