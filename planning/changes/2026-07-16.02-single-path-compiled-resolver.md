# Design: Single-path compiled-closure resolver

## Summary

Replace modern-di's interpreted resolve recursion (`Container.resolve_provider`
→ `Factory.resolve` → `_resolve_kwargs` → recurse) with **one** compiled
**closure** resolver per provider, built lazily on first resolve and memoized on
`ProvidersRegistry` exactly like the wiring plan. There is no second path: no
`exec`, no interpreted fallback, no runtime switch. Each resolver bakes its branch
decisions and calls its dependencies' resolvers by reference (≈1 frame/node), so
the per-node framework tax — scope navigation, override lookup, closed-state
check, kwargs assembly — is hoisted out of the per-call path. Target: at or below
dishka on the comparative benchmark, with behavior-equivalence as the hard gate.

## Motivation

The resolve path is 3-6x slower than dishka (338/231/615 ns transient / warm
singleton / depth-6 chain vs our 1064/411/3280; change `2026-07-15.05`). The
A1-A4 micro-opts (`2026-07-15.06`) shaved only 5.7-9.2%. A 2026-07-16 spike proved
a compiled **closure** resolver lands *below* dishka on all three graphs
(155/64/464). The first attempt to make it real — the additive fast path
(`2026-07-16.01`) — reached only 30-43% because it reused `Factory`'s methods and
kept the interpreted path alongside, so it paid the full per-node tax from a
compiled path; worse, being additive it carried permanent complexity (a fallback,
a `_force_interpreted` module global in a no-global-state library, a shipped
differential harness, and two copies of the resolve logic). The competitor
research ([audit 2026-07-16](../audits/2026-07-16-competitor-perf-research-report.md))
settles the design: the ≈2x win is *hoisting per-node decisions to build time*,
closures capture ~80-90% of the ceiling, and `exec` buys 0-4% at fixed arity — so
the clean move is a **single** compiled path, not an additive one.

## Design

**Mechanism.** `ProvidersRegistry.resolver_for(provider)` builds and memoizes a
`Callable[[Container], T]` keyed by `(provider_id, version)` — identical lifetime
and invalidation to the wiring-plan memo. `resolve_provider` collapses to:

```python
def resolve_provider(self, provider):
    return self.providers_registry.resolver_for(provider)(self)
```

The interpreted `Factory.resolve`/`_resolve_kwargs` bodies are **deleted** (they
survive only behind a dev-time test flag; see Testing). A resolver calls each
dependency's resolver **by reference** (one frame per node — the spike's model,
not `exec`-inlined to zero frames), which keeps overrides and cycles simple.

**All four provider types compile** (no fallback remains): `Factory` is the
substantive one; `Alias` becomes literally its target's resolver (zero
indirection); `ContainerProvider` returns the navigated container;
`ContextProvider` does the context lookup.

**Hoisting the per-node tax:**

- **Scope navigation** (kills `find_container` per node): a resolver takes its
  target as `container if container._scope == baked_scope else
  container.find_container(baked_scope)` and passes that target down. Same-scope
  deps — the common case and the whole benchmark chain — hit an int compare and
  **skip the call**; only genuinely cross-scope edges navigate. Same-scope chains
  navigate once, at the root.
- **Overrides**: a live `if overrides_registry.any:` guard. Production steady
  state pays one bool load per node and never touches the dict; runtime
  `override()`/`reset_override()` stay fully correct with **no** resolver
  invalidation.
- **Closed-state check** runs once per distinct target container, not per node.
- **Positional creator calls** replace `creator(**kwargs)` where the wiring plan
  shows positional-compatible params (fixing the measured 4-6x `**kwargs` cost);
  context/static/keyword-only args still go by name.
- **Transient vs cached split**: transient resolvers carry no cache machinery at
  all; cached resolvers keep the double-checked warm-hit + `get_or_create(lock)` +
  `mark_created` (locking and LIFO finalizers reused unchanged).

**Semantics preserved by reuse, not reimplementation.** Each closure keeps a
per-node `try/except` and reuses `_resolution_step`/`prepend_step` for
byte-identical breadcrumbs (the 1-frame-per-node model preserves the call-stack
chain). Cross-scope errors, cache locking, and LIFO finalizers are reused. Cycles:
build-time back-edge detection during resolver construction plus the runtime
`RecursionError → CircularDependencyError` guard, **carrying over** cut-1's
canonical cycle rooting (`dependency_graph.py`) and self-contained cycle errors
(`exceptions.py`) — the two strictly-better improvements from the abandoned branch.

**Promotion:** `architecture/resolution.md` is rewritten to describe the compiled
path; `containers.md`/`providers.md` are touched where they reference the
interpreted recursion — in the same PR.

## Non-goals

- **`exec`/codegen** — rejected: closures ≈ exec (0-4% at fixed arity) and `exec`
  in a zero-dependency library is unjustified (audit §1/§4).
- **Additive fast path with interpreted fallback** — this is what `2026-07-16.01`
  did; superseded here.
- **Whole-aggregate warm-singleton memoization** (wireup's depth-independent warm
  hit) — deferred; the per-provider cache suffices. Add only if benchmarks demand.
- Any public API change, new dependency, or change to resolution semantics.
- Compiling scope/override handling to build-time-only (dishka's model) — modern-di
  keeps a runtime `override()` test API.

## Testing

- **Dev-time differential scaffold (deleted before merge).** Keep today's
  interpreted resolver alive behind a test-only flag and reuse cut-1's A/B harness
  to assert the closure resolver produces identical return values **and** identical
  raised-exception `str()` across the full battery (transient, cached, deep chain,
  wide, cross-scope, overrides, context deps, nullable/static kwargs, unwireable
  errors, cycles). Once green, **delete the interpreted path, the flag, and the
  harness**; the shipped tree has exactly one resolve path.
- `just test-ci` green with the compiled path at **100% line coverage**;
  `just lint-ci` green (also validates this bundle).
- **Perf (paired).** Guard tier before/after by the paired interleaved method
  (`2026-07-15.06`); comparative tier re-measured vs dishka on C1-C3. Goal is
  at-or-below dishka; the hard gate is behavior-equivalence + no guard-tier
  regression, not a specific dishka margin.

## Risk

- **Behavior drift with no permanent second path** (med × high) — the dev-time
  differential scaffold plus 100% coverage is the mitigation; the real risk is a
  *gap in the battery*, so it must exercise every semantic in the Design table
  before the scaffold is deleted.
- **Breadcrumb `str()` mismatch** (med × med) — `prepend_step`/`_resolution_step`
  are reused, not reimplemented; the harness asserts exact `str()`.
- **Cycle/recursion edge behavior** (low × high) — carry cut-1's canonicalization
  and self-contained cycle errors verbatim; covered by existing cycle tests and
  the harness.
- **Integrated perf short of the spike** (low × med) — the spike omitted mandatory
  per-node semantics; the at-or-below-dishka goal (not spike parity) absorbs it,
  and an honest shortfall is a maintainer data point, not a silent pass.
