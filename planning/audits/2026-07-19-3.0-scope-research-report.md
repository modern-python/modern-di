## cleanup

### UnsupportedCreatorParameterError for positional-only params doesn't surface the simpler fixes
- Lens: ux-api-dx
- Surface: `modern_di/types_parser.py:67-74`, `modern_di/exceptions.py` (`UnsupportedCreatorParameterError`)
- Leverage / Risk: medium / low   ·   Confidence: high   ·   3.0: non-breaking

**What.** When a creator has a positional-only parameter with no default, the error offers two fixes as equals: "give the parameter a default or use `skip_creator_parsing=True`". But `skip_creator_parsing` disables signature introspection entirely and emits a warning unless `bound_type` is set explicitly — a heavy lift presented as peer to a one-line default. A third, equally-cheap fix (make the parameter keyword-only) is documented in troubleshooting but absent from the error text.

**Evidence.**
```
raise exceptions.UnsupportedCreatorParameterError(
    reason='positional-only parameters cannot be passed by keyword; '
           'give the parameter a default or use skip_creator_parsing=True'
)
# skip_creator_parsing (factory.py:71-76) requires bound_type and warns if omitted — nuclear option
# keyword-only params (types_parser.py:84-87) resolve fine — the cheap fix the message omits
```

**Direction.** Reorder and complete the suggestion hierarchy: (1) add a default, (2) make the parameter keyword-only, (3) `skip_creator_parsing=True` as last resort. Error text is not a contract (per the `error-text-is-not-a-contract` decision), so this is a free change; no prior candidate covers this specific message.

### ContextProvider `scope=UNSET` default is under-explained in the reference docs
- Lens: ux-api-dx
- Surface: `modern_di/providers/context_provider.py:25-35`, `docs/providers/context.md`
- Leverage / Risk: low / low   ·   Confidence: medium   ·   3.0: non-breaking

**What.** `ContextProvider` takes `scope: enum.IntEnum | types.UnsetType = types.UNSET`. `UNSET` resolves to the Group's scope, else `Scope.APP` (via `AbstractProvider.__init__` and `Group.__init_subclass__` stamping). But `docs/providers/context.md` claims the default is `Scope.APP` and never explains `UNSET` semantics or when to override. Factory's reference docs carry this note (`architecture/providers.md:56`); ContextProvider's do not. A developer may redundantly set `scope=Scope.REQUEST` when the enclosing Group already defaults to it.

**Evidence.**
```
scope: enum.IntEnum | types.UnsetType = types.UNSET   # context_provider.py:29
# abstract.py:25-26: self.scope = Scope.APP if isinstance(scope, UnsetType) else scope
# group.py:28-30: group-level scope stamps UNSET providers
# docs/providers/context.md:13 claims scope=Scope.APP — omits UNSET + group inheritance
```

**Direction.** Add a "scope parameter" note to `docs/providers/context.md` mirroring Factory's: "Defaults to `UNSET` — inherits the Group's scope, else `Scope.APP`. Override only when a provider needs a scope different from its group default." Fresh, orthogonal to the DOC-2/DOC-4 quickstart work per the prior-art lens.
